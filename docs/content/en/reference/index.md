# Reference

look things up; they describe *what is*, not *how to* (for tasks, see the
[how-to guides](../how-to/index.md)).

- [CLI](cli.md) - every command and its options.
- [Configuration](configuration.md) - the `Config` object and the presets.
- [Frontmatter](frontmatter.md) - the per-document metadata fields PySSG reads.
- [Plugins and hooks](plugins.md) - the built-in and contrib plugins, and the hook
  points they tap.
- [API reference](/references/pyssg/) - the auto-generated docstring reference
